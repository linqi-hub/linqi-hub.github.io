    # 7.盒模型

    盒模型是由content、padding、border、margin构成，分为标准盒模型和怪异盒模型，区别在于width和height的范围不同。

    • 标准盒模型：width/height = content
    • 怪异盒模型：width/height = content + padding + border

    通过box-sizing属性来设置：

    box-sizing:content-box   // 默认值，为标准盒模型
box-sizing:border-box    // 怪异盒模型

    ####8.定位

    一种更加高级的布局手段，通过定位可以将元素摆放到页面的任意位置。

    position的属性值：

    属性值 含义
    static 不开启定位（默认值）
    relative 相对定位
    absolute 绝对定位
    fixed 固定定位
    sticky 粘滞定位

    （1）relative

    • 开启相对定位以后，不设置偏移量，元素位置不会发生任何变化（形神合一）
    • 参照自身原始位置进行定位；
    • 会提升元素的层级（可以覆盖其他元素）；
    • 不会改变元素的性质（块还是块，行还是行）；

    （2）absolute

    • 开启绝对定位以后，元素会从文档流中脱离；
    • 会改变元素的性质（行内变成块，块的宽高被内容撑开）；
    • 会提升元素的层级；
    • 参照最近的开启定位的元素进行偏移；

    （3）fixed

    固定定位是一种特殊的绝对定位，只不过他永远参照浏览器的视口进行定位。

    （4）sticky

    与相对定位的特点基本一致。

    • 该元素是根据文档流进行定位的，即相对于包含块进行偏移
    • 偏移量不会影响任何其他元素的位置
    • 粘性元素总是“粘”到其最近的具有“滚动机制”的祖先元素（当overflow为hidden、scroll、auto、overlay时创建），即使该祖先不是最近的实际滚动祖先

    z-index只有作用于定位元素时才会生效。

    # 4.长度单位

    （1）像素（px）：显示器屏幕是由一个一个小点构成的。

    （2）百分比：相对于父元素属性设置的比例。

    （3）em：相对于元素的字体大小来计算。

    （4）rem：相对于根元素的字体大小来计算。

    # 5.颜色单位

    （1）RGB（red，green，blue），每一种颜色的范围都在0~255之间。

    （2）RGBA，在RGB的基础上增加了一个不透明度a。

    （3）十六进制的RGB，就是RGB的十六进制写法，每一种颜色的范围都在00~ff之间。

    # 6.文档流

    我们所创建的元素默认都是在文档流中进行排列的，对于元素来说有两种状态：

    • 在文档流中
    • 脱离文档流

    （1）块元素

    • 块元素在页面中独占一行
    • 默认宽度是父元素的全部（会把父元素撑满）
    • 默认高度是内容撑开（子元素的高度）

    （2）行内元素

    • 行内元素不独占一行，根据自身大小占据
    • 从左向右水平排列
    • 排满后自动换行
    • 默认宽度和高度都是被内容撑开的

    # 7.盒模型

    CSS将页面中的元素都设置成为一个矩形的盒子，布局就可以把不同的盒子放到不同的位置。

    所以边框和内边距的大小都会影响盒子的大小。边距影响盒模型的位置。

    （1）content

    通过width和height来设置内容区的大小。

    （2）padding

    （3）border

    当给边框设置属性时，值决定的是上、右、下、左。

    （4）margin

    元素在页面中是按照自左向右的顺序排列的，所以默认情况下

    • 设置左和上外边距：移动元素自身
    • 设置下和右外边距：移动其他元素

    # 8.水平方向布局

    元素在水平方向的布局必须满足以下等式：

    margin-left + border-left + padding-left + width + padding-right + border-right + margin-right = 其父元素的宽度

    以上等式必须满足，如果相加结果使等式不成立，则称为过渡约束，然后浏览器会自动调整左边数值，让其满足等式。

    • 没有auto：浏览器会自动调整margin-right值以使等式满足
    • 有auto：则会自动调整auto值以使等式成立 这七个值中有三个值可以设置为auto ：width、margin-left、maring-right

    # 9.垂直方向布局

    （1）元素溢出

    子元素是在父元素的内容区中排列的，如果子元素的大小超过了父元素，则子元素会从父元素中溢出。 使用overflow属性来设置父元素如何处理溢出的子元素：

    • visible：默认值，溢出内容会在父元素外部位置显示。
    • hidden：溢出内容会被裁剪掉，不会显示。
    • scroll：生成两个滚动条，通过滚动条来查看完整的内容。
    • auto：根据内容需要，自动生成滚动条。

    （2）外边距重叠

    相邻的垂直方向外边距会发生重叠：

    • 两者都是正值：取较大的
    • 一正一负：取两者的和
    • 都是负值：取绝对值较大的

    # 10.行内元素盒模型

    • 不支持设置宽度和高度
    • 可以设置padding，但是垂直方向的padding不会影响页面的布局
    • 行内元素可以设置border，垂直方向的border不会影响页面的布局
    • 行内元素可以设置margin，垂直方向的margin不会影响页面的布局

    如果想要把一个a链接，设置宽高：

    # 12.浮动

    通过浮动可以使一个元素向其父元素的左侧或右侧移动。

    使用float属性来设置元素的浮动：

    • none：元素不浮动；
    • left：元素向左浮动；
    • right：元素向右浮动；

    浮动的特点：

    • 浮动元素会完全脱离文档流，不再占据文档流中的位置；
    • 浮动元素默认不会从父元素中移除；
    • 浮动元素向左或向右移动时，不会超过前边的浮动元素（先来后到）；
    • 浮动元素不会超过上边的浮动元素的兄弟元素，最多就是和它一样高；
    • 如果浮动元素的上边是一个没有浮动的块元素，则浮动元素无法上移；
    • 浮动元素不会盖住文字，文字会自动环绕在浮动元素的周围；

    脱离文档流的特点：

    • 块元素不再独占页面的一行；
    • 脱离文档流以后，块元素的宽度和高度默认都被内容撑开；

    # 13.高度塌陷与BFC

    高度塌陷：

    在浮动布局中，父元素的高度默认是被子元素撑开的，当子元素浮动后，会完全脱离文档流，子元素从文档流中脱离将无法撑起父元素的高度，导致父元素的高度丢失。

    BFC（block formatting context）块级格式化上下文：

    bfc是一个隐含的属性，可以为一个元素开启BFC，开启bfc之后该元素会变成一个独立的布局区域。

    BFC元素的特点：

    • 不会被浮动元素覆盖；
    • 父子元素外边距不会重叠；
    • 可以包含浮动的元素；

    开启BFC的方式：

    • 设置为浮动；
    • 设置为行内块元素；
    • 设置overflow为hidden；
    • 绝对定位元素；
    • 文档根元素；

    # 14.清除浮动

    当我们不希望某个元素因为其他元素浮动的影响而改变位置，可以通过clear属性来清除浮动元素对当前元素所产生的影响。

    clear可选值：

    • left 清除左侧浮动元素对当前元素的影响
    • right 清除右侧浮动元素对当前元素的影响
    • both 清除两侧中影响较大一侧元素的影响（注意，这里不是同时清除两侧的影响）

    解决外边距重叠问题：

    # 15.定位

    通过定位可以将元素摆放到页面的任意位置，定位是一种更加高级的布局手段。

    position的属性值：

    属性值 含义
    static 不开启定位（默认值）
    relative 开启相对定位
    absolute 开启绝对定位
    fixed 开启固定定位
    sticky 开启粘滞定位

    （1）relative

    • 开启相对定位以后，不设置偏移量，元素位置不会发生任何变化（形神合一）
    • 参照自身原始位置进行定位；
    • 会提升元素的层级（可以覆盖其他元素）；
    • 不会改变元素的性质（块还是块，行还是行）；

    （2）absolute

    • 开启绝对定位以后，元素会从文档流中脱离；
    • 会使元素提升一个层级；
    • 会改变元素的性质（行内变成块，块的宽高被内容撑开）；
    • 会提升元素的层级；
    • 参照最近的开启定位的元素进行定位；

    （3）fixed

    固定定位是一种特殊的绝对定位，只不过他永远参照浏览器的视口进行定位，不会随网页的滚动条滚动。

    （4）sticky

    与相对定位的特点基本一致。

    • 该元素是根据文档流进行定位的，即相对于包含块进行偏移
    • 偏移量不会影响任何其他元素的位置
    • 粘性元素总是“粘”到其最近的具有“滚动机制”的祖先元素（当overflow为hidden、scroll、auto、overlay时创建），即使该祖先不是最近的实际滚动祖先

    对于开启了定位元素，可以通过z-index属性来指定元素的层级。

    # 17、动画

    动画和过渡类似，都是可以实现一些动态的效果，不同的是:

    • 过渡需要在某个属性发生变化时才会触发
    • 动画可以自动触发动态效果

    设置动画效果，必须先要设置一个关键帧，关键帧设置了动画执行每一个步骤

    @keyframes test {
    from {
        margin-left: 0;
    }
    to {
        margin-left: 900px;
    }
}

    • normal 从from向to运行，每次都是这样，默认值
    • reverse 从to向from运行，每次都是这样
    • alternate 从from向to运行，重复执行动画时反向执行
    • alternate-reverse 从to向from运行，重复执行动画时反向执行

    animation-play-state 设置动画的执行状态

    • running 动画执行，默认值
    • paused 动画暂停
    • animation-fill-mode 动画的填充模式
    • none 动画执行完毕，元素回到原来位置，默认值
    • forwards 动画执行完毕，元素会停止在动画结束的位置
    • backwards 动画延时等待时，元素就会处于开始位置
    • both 结合了forwards和backwards
    <style>
  .box1 {
    width: 256px;
    height: 256px;
    background-image: url(./bg2.png);
    animation: run 1s steps(6) infinite;
  }
  @keyframes run {
    from {
      background-position: 0 0;
    }
    to {
      background-position: -1536px 0;
    }
  }
</style>
