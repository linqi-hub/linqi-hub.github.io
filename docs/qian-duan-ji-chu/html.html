    lynch

    # 1. src和href的区别

    都是对外部资源的引用。

    区别：

    • src（source）：资源引用，引用的内容会嵌入到当前标签所在的位置，下载该资源时会暂停其他资源的处理，知道引用资源加载、编译执行完毕，所以js脚本一般放在页面底部。
    • href（Hypertext Reference）：超文本引用，引用的内容指向一些网络资源，建立和当前元素或文本的链接关系，会与其他资源并行下载，常用<a>、<link>等标签。

    # 2.HTML语义化

    语义化就是根据内容选择合适的HTML标签。

    优点：

    • 对浏览器更友好，有利于SEO；
    • 代码结构清晰，可读性强，便于开发和维护；

    语义化标签：

    • <header>--头部
    • <nav>--导航栏
    • <section>--区块
    • <article>--主要内容
    • <aside>--侧边栏
    • <footer>--底部

    # 3.DOCTYPE

    文档声明类型，用来告知浏览器以何种文档类型定义（HTML和XHTML）解析文档。如果不写DOCTYPE浏览器就会以怪异模式对页面进行渲染。

    • 标准模式：浏览器使用W3C的标准解析渲染页面，浏览器使用它支持的最高标准呈现页面。
    • 怪异模式：浏览器使用自己的怪异模式解析渲染页面，页面以一种宽松的向后兼容方式显示。

    # 4.defer和async

    两者都是异步加载外部JS脚本文件，并且不会阻塞页面的解析。

    图中绿色为HTML解析，灰色为HTML解析被打断；紫色为script下载；分红为script执行；

    1. <script>:

    HTML解析被中断，script下载并且执行完毕继续解析HTML。

    1. <script async>:

    script与HTML解析并行，但script执行时中断HTML解析。

    1. <script defer>:

    script与HTML解析并行，并且script在HTML解析完毕后开始执行。

    # 5.meta标签

    meta标签存在于head标签中，用来描述网页文档的属性。

    meta有个必须的属性content用于表示需要设置的项的值。

    meta存在两个非必须的属性http-equiv和name, 用于表示要设置的项。

    #####1.http-equiv 属性

    http请求头相关的信息，设置的值会关联到http头部。浏览器在请求服务器获取html的时候，服务器会将html中设置的meta放在响应头中返回给浏览器。

    • content-type

    比如：<meta http-equiv="content-type" content="text/html charset=utf8">可以用来声明文档类型、设字符集，目前content-type只能在html文档中使用。

    这样设置浏览器的头信息就会包含:

    content-type: text/html charset=utf8
    1
    • expires

    用于设置浏览器的过期时间, 其实就是响应头中的expires属性。

    <meta http-equiv="expires" content="31 Dec 2021">
    1
    • refresh

    该种设定表示5秒自动刷新并且跳转到指定的网页。如果不设置url的值那么浏览器则刷新本网页。

    <meta http-equiv="refresh" content="5 url=http://www.qianduan.com">
    1
    • window-target

    强制页面在当前窗口以独立页面显示, 可以防止别人在框架中调用自己的页面。

    <meta http-equiv="window-target" content="_top'>
    1

    #####2.name 属性

    name属性主要用于描述网页，name设置属性名，content设置属性值。

    • author

    author用来标注网页的作者

    <meta name="author" content="aaa@mail.abc.com">
    1
    • description

    description用来告诉搜素引擎当前网页的主要内容，是关于网站的一段描述信息。

    <meta name="description" content="这是我的HTML">
    1
    • keywords

    keywords设置网页的关键字，来告诉浏览器关键字是什么。是一个经常被用到的名称。它为文档定义了一组关键字。某些搜索引擎在遇到这些关键字时，会用这些关键字对文档进行分类。

    <meta name="keywords" content="Hello world">
    1
    • robots

    告诉搜索引擎机器人抓取哪些页面，all / none / index / noindex / follow / nofollow。

    <meta name="robots" content="all">
    1

    all：文件将被检索，且页面上的链接可以被查询；none：文件将不被检索，且页面上的链接不可以被查询；index：文件将被检索；follow：页面上的链接可以被查询；noindex：文件将不被检索，但页面上的链接可以被查询；nofollow：文件将不被检索，页面上的链接可以被查询。

    # 6.HTML5的更新

    • 语义化标签

    • 媒体标签：音频audio、视频video

    • 表单

      表单类型：email、url、number、time、week、month

      表单属性：placeholder、required、multiple、autofocus

    • DOM查询操作

      两个支持使用选择器查找元素的api，可以是类（.），可以使id（#）：

      document.querySelector()

      document.querySelectorAll()

    • Web存储

      localStorage

      sessionStorage

    • 拖拽

      • 选中
      • 拖动：dragstart--drag--dragend(被拖动的元素)、dragenter--dragover--dragover（目的地对象）
      • 释放：drop
    <img draggable='true | false | auto'>
    1

    • 画布

      用于在网页绘制2D图形和图像

    # 7.行内元素、块级元素和空元素

    • 行内元素

      与其他行内元素存在一行、不可设置宽高

      a、b、span、img、input、select、strong

    • 块级元素

      独占一行，可以设置宽、高

      div、ul、ol、li、h、p、form、table

    • 空元素

      没有内容的HTML元素，自闭和标签。

      br、hr、img、input、link、meta、source

    # 8.web worker

    在HTML页面中，执行脚本时，页面的状态是不可相应的，直到脚本执行完成后，页面才变成可以相应的。

    web worker是运行在后台的js，独立于其他脚本，不会影响页面的性能，通过PostMessage将执行结果返回给主线程，执行复杂操作的时候不会影响主线程。

    • 检测浏览器对于web worker的支持性；
    • 创建web worker文件；
    • 创建web worker对象；

    # 9.Canvas和SVG的区别

    • canvas：画布，通过JavaScript来绘制2D图形，逐像素进行渲染，位置发生改变便重新渲染；
    • svg：可缩放矢量图，基于可扩展XML语言描述2D图形，DOM中的每个元素都是可用的；
    canvas svg
    依赖分辨率 不依赖分辨率
    不支持事件处理 支持事件处理
    能够保存png、jpg格式的图片 过度使用DOM的应用渲染速度慢
    适合于图像密集型的游戏 适合谷歌地图
