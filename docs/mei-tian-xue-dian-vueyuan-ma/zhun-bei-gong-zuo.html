    # 认识Flow

    Flow 是 facebook 出品的 JavaScript 静态类型检查⼯具。Vue.js 的源码使用 Flow 进行静态类型检查。

    JavaScript 是动态类型语言，使用灵活，但有些代码的错误过于隐蔽，编译时候不会报错，运行时就会出现各种奇怪的bug。

    类型检查是动态类型语言的发展趋势，通过在编译期尽早发现（由类型错误引起的）bug，不影响代码的运行，使编写的JavaScript代码和强类型语言相似。

    项目越复杂就需要通过工具来保证项目的可维护性和代码的可读性，Vue在2.0重构时，通过ESLint来保证代码的风格，通过Flow确定类型检查(flow对于Babel和ESLint都有对应的插件支持，使用成本小)。

    # Flow的工作方式

    工作方式分为两种：

    • 类型推断：通过变量的使用环境来推断出变量的类型，根据推断结果进行类型检查。
    • 类型注释：通过注释的类型进行类型检查。

    类型推断：

    /@flow/``
function split(str) {
	return str.split(' ')
}
split(11)
// Flow 检查上述代码后会报错，因为函数split需要的是字符串。

    # Flow在源码中的应用

    引用第三方库或者自定义类型，Flow不认识，此时Flow通过libdef来识别，Vue.js主目录下面的.flowconfig文件包含flow的配置

    ├── compiler.js # 编译相关 ├── component.js # 组件数据结构 ├── global-api.js # Global API 结构 ├── modules.js # 第三⽅库定义 ├── options.js # 选项相关 ├── ssr.js # 服务端渲染相关 ├── vnode.js # 虚拟 node 相关

    Vue.js 有很多自定义类型，可以翻阅来确定这些数据结构的定义。

    # Vue.js源码目录结构设计

    Vue.js的源码都在src目录下面，结构如下：

    src ├── compiler # 编译相关 ├── core # 核心代码 ├── platforms # 不同平台的支持 ├── server # 服务端渲染 ├── sfc # .vue 文件解析 ├── shared # 共享代码

    1. compiler

      Vue.js所有编译相关的代码，包括把模板解析成ast语法树、ast语法树优化、代码生成。

      编译可以构建时进行（通过webpack、vue-loader插件），也可以使用包含构建功能的Vue.js，编译是一项耗性能的工作，所以更难推荐离线编译。

    2. core

      Vue的核心代码，包括内置组件、全局APi封装、Vue实例化、观察者、虚拟DOM、工具函数等。Vue的灵魂都在这里

    3. platform

      Vue是跨平台的MVVM框架，可以在Web上运行，也可以配合weex在native客户端上运行，platform是Vue的入口，两个目录代表两个主要入口，打包运行在web和weex上面。

    4. server

      Vue服务端渲染相关的逻辑都在这个目录下面，这部分代码是运行在服务端的node.js上。

      服务端渲染：把组件渲染为服务端的HTML字符串，然后发送到浏览器，最后将静态标记“混合”为客户端上完全交互的应用程序。

    5. sfc

      把.vue文件内容解析成一个JavaScript对象。

    6. shared

      Vue定义的工具和方法，被浏览器和服务器共享。

    作者把功能模块拆解的非常清楚，相关的逻辑放到独立的目录下面，复用的代码抽成独立目录，增强了代码的阅读性和可维护性。

    # Runtime Only VS Runtime+Compiler

    1. Runtime Only

      该版本的Vue.js通常需要借助webpack的vue-loader把.vue文件编译成为JavaScript，因为是在编译阶段做的，所以只包含运行时的Vue.js代码，因此代码的体积会更轻量。

    2. Runtime+Compileer

      在客户端编译模板。

    // 需要编译器的版本
new Vue({
	template: '{{ hi }}'
})

// 这种情况不需要
new Vue({
	render (h) {
		return h('div', this.hi)
	}
})

    # initGlobalAPI

    Vue初始化的时候不仅给它的原型prototype上扩展方法，还给Vue这个对象本身扩展全局静态方法，定义在 src/core/global-api/index.js 中：

    export function initGlobalAPI (Vue: GlobalAPI) {
  // config
  const configDef = {}
  configDef.get = () => config
  if (process.env.NODE_ENV !== 'production') {
    configDef.set = () => {
      warn(
      'Do not replace the Vue.config object, set individual fields instead.'
      )
    }
  }
	Object.defineProperty(Vue, 'config', configDef)
  // exposed util methods.
  // NOTE: these are not considered part of the public API - avoid relying on
  // them unless you are aware of the risk.
  Vue.util = {
    warn,
    extend,
    mergeOptions,
    defineReactive
  }
  Vue.set = set
  Vue.delete = del
  Vue.nextTick = nextTick
  Vue.options = Object.create(null)
  ASSET_TYPES.forEach(type => {
  	Vue.options[type + 's'] = Object.create(null)
  })
  // this is used to identify the "base" constructor to extend all plain-object
  // components with in Weex's multi-instance scenarios.
  Vue.options._base = Vue
  extend(Vue.options.components, builtInComponents)
  initUse(Vue)
  initMixin(Vue)
  initExtend(Vue)
  initAssetRegisters(Vue)
}
