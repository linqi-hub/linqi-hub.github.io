    # Vue简介

    用于构建用户界面的渐进式JavaScript框架，Vue可以自底向上逐层的应用。 简单应用只需要一个轻量小巧的核心库。 复杂应用引入各式各样的Vue插件。

    • 13年尤雨溪开发，12月发布版本号0.6。
    • 15年10月27，正式发布1.0。
    • 16年10月1日，正式发布2.0。
    • 20年9月18日，正式发布3.0。

    # Vue的特点

    • 采用组件化模式，提高代码的复用率，代码更容易维护。
    • 声明式编码，不用直接操作DOM，提高开发效率。
    • 使用虚拟DOM + Diff算法，复用DOM节点。

    # 数据代理

    1. MVVM模型
    • M：模型(Model)，data中的数据；
    • V：视图(View)，模板代码；
    • VM：视图模型(ViewModel)，Vue实例；
    1. 数据代理

    通过一个对象代理另一个对象中属性的操作(读/写) Vue中的数据代理：

    • 通过vm对象来代理data对象中属性的操作
    • 更加方便的操作data中的数据

    基本原理：

    • 通过Object.defineProperty()把data对象中所有属性添加到vm上；
    • 为每一个添加到vm上面的属性都指定一个getter和setter；
    • 在getter和setter内部去操作data中对应的属性；

    # 事件

    1. 基本使用

      • 使用v-on:xxx或@xxx来绑定事件；
      • 事件的回调需要在methods中配置，最终会在vm上，被vue管理，this指向vm或者组件实例对象；
      • methods中配置的函数不要用箭头函数，否则this不是vm了；
      • 如果需要传参数的话直接在回调函数中配置即可；

    2. 事件修饰符

      • prevent:阻止默认事件；
      • stop：阻止事件冒泡；
      • once：事件只触发一次；
      • capture：使用事件的捕获模式；
      • self：只有event.target是当前操作的元素是才触发事件；
      • passive：事件立即执行，无需等待事件回调执行完毕；

    修饰符可以连续使用，按功能叠加即可。

    // 默认是在回调函数上添加e.preventDefault()
// 现在只需要使用事件修饰符，添加prevent即可

    # 使用watch实现

    watch，侦听器，直接对已有属性进行监听即可，watch不需要返回值。

    watch默认不监测对象内部值得变化(一层)，配置deep:true可以检测对象内部值得改变(多层)。

    Vue默认可以检测内部值得改变，所以使用watch时根据数据的结构，决定是否采用深度监视。

    watch: {
  firstName(val) {
    this.fullName = val + '-' + this.lastName
  },
  lastName(val) {
    this.fullName = this.firstName + '-' + val
  }
}

    # 列表渲染

    关键点：

    • 需要建立一个新的对象来保存处理后的结果；
    • 使用v-for遍历处理后的结果；
    • 然后再输入框内使用v-model来确定关键词的筛选；
    // 结构
<div id="root">
  <input type="text" placeholder="请输入姓名" v-model='keyWords'>
  <button @click='sortType = 2'>年龄升序</button>
  <button @click='sortType = 1'>年龄降序</button>
  <button @click='sortType = 0'>原顺序</button>
  <ul>
    <li v-for='p of filPersons' :key='p.id'>
      {{p.name}} -- {{p.age}}
    </li>
  </ul>
</div>
// 样式
new Vue({
  el: '#root',
  data: {
    persons: [{
      id: '001',
      name: '马冬梅',
      age: 19
    }, {
      id: '002',
      name: '周冬雨',
      age: 20
    }, {
      id: '003',
      name: '周杰伦',
      age: 21
    }, {
      id: '004',
      name: '温兆伦',
      age: 22
    }],
    keyWords: '',
    sortType: 0
  },
  computed: {
    filPersons() {
      const arr = this.persons.filter((p) => {
        // 字符串中indexOf('') == 0,所以刚开始就能返回一个完整的对象
        return p.name.indexOf(this.keyWords) != -1
      })
      if (this.sortType) {
        arr.sort((p1, p2) => {
          return this.sortType == 1 ? p2.age - p1.age : p1.age - p2.age
        })
      }
      return arr  // 最终要返回一个结果
    }
  }
})

    # key的作用

    key是虚拟对象的标识，当数据发生变化时，可以使用key快速拿到新旧虚拟DOM进行对比，新旧虚拟DOM中key不同的话，直接创建新的真实DOM，如果key值相同，则比较虚拟DOM中的内容。

    最好使用每条数据的唯一标识作为key，如id、手机号、身份证号等。

    使用index作为key的问题：对数据进行逆序删除添加等操作，会产生没必要的真实DOM更新，效率低，如果结构中包含输入类的DOM，会产生错误的DOM更新。

    # Vue监视数据的原理

    vue会监视data中所有层次的数据，对data中的所有数据进行加工，使其变成响应式。

    1. 检测对象中的数据

    通过setter实现监视，在new Vue时传入要检测的数据。

    模拟一个数据检测：

    // 普通的数据
const data = {
  name = 'lynch',
  address = 'beijing',
}

// 创建一个监视的实例对象，用于监视data中属性的变化
const obs = new Observer(data)

// 准备一个vm实例对象
let vm = {}
vm._data = data = obs

// 数据代理
function Observer(obj){
  // 收集对象中的所有属性
  const keys = Object.keys(obj)
  // 遍历
  keys.forEach(k=>{
    Object.defineProperty(this,k,{
      get(){
        return obj[k]
      },
      set(val){
        obj[k] = val
      }
    })
  })
}

    # 生命周期

    Vue在关键时刻帮我们调用一些生命周期函数。

    常用的生命周期钩子：

    • mounted：发送ajax请求、启动定时器、绑定自定义事件、订阅消息等(初始化工作)。
    • beforeDestroy：清除定时器、解绑自定义事件、取消订阅消息等(首尾工作)。

    关于销毁Vue实例

    • 销毁Vue实例后，自定义事件会失效，但原生DOM事件依然有效。
    • 一般不要再beforeDestroy操作数据，因为即便操作数据，也不会触发更新流程了。

    # Vue中使用组件的三大步骤

    • 定义组件
    • 注册组件
    • 使用组件
    1. 定义组件

    使用Vue.extend(options)创建组件。

    注意点：

    组件创建的时候不能配置el选项，因为所有的组件都要被一个vm管理，在vm中决定服务哪个容器。

    组件中的data必须写成函数，因为对象是引用数据类型，写成函数，可以避免组件被复用的时候数据存在引用关系。

    1. 注册组件
    • 全局注册：通过Vue.component('组件名'，组件)。
    • 局部注册：在new Vue的时候传入components选项。
    1. 使用组件

    直接使用组件名作为标签即可。

    // 1.创建组件school
const school = Vue.extend({
  template:`
    <div>
      <h2>学校名称<h2/>
      <h2>学校地址<h2/>
    <div/>
  `,
  data(){
    return{
      schoolName:'北京大学',
      address:'北京'
    }
  }
})

new Vue({
  el:'#app',
  data:{
    msg:'你好啊'
  },
  // 2.局部注册school组件
  components:{
    school,
  }
})
// 3.使用school组件
 <div>
   <h1>msg<h2/>
   <school>学校地址<school/>
 <div/>

    # 组件的注意点

    1. 组件名为单个词时，首字母大小写都可以，当为多个单词的时候，可以使用my-school的写法，如果使用驼峰式写法，需要脚手架的支持
    2. 以下两种方式都可以：
    const school = Vue.extend(options)
const school = options
    1
    2
    1. 当多个组件发生嵌套的时候，组件在谁那里注册，就在谁那里使用，不注册就不能使用。

    # VueComponent

    组件的本质就是一个名为VueComponent的构造函数，是Vue.extend生成的，并且每次通过Vue.extend去创建一个组件的时候，都会返回一个全新的VueComponent。

    所以使用组件的时候就会调用VueComponent这个构造函数，Vue解析标签，帮我们创建一个组件的实例对象。(注意，Vue的实例对象是vm,vm可以有el这个选项，而组件实例对象不能有)

    # 一个重要的内置关系

    VueComponent.prototype.__proto__  === Vue.prototype
    1

    Vue为什么要有这个内置关系：因为让组件的实例对象可以访问到Vue原型上的属性和方法。
