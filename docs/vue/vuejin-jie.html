    # 脚手架文件结构

    可以直接使用脚手架搭建我们的工程，项目目录如下：

    ├── node_modules 
	├── public

    # 残缺版Vue和完整版Vue

    通过脚手架创建初始化Vue项目时，引入的是运行版的Vue，.vue.runtime.xxx.js，他只包含核心功能，没有模板解析器，所以main.js中创建Vue实例对象需要使用render函数，解析vm中的模板。

    new Vue({
  el:'#app',
  render:h => h(App)
})

// render函数为下面的简写
render(App){
  return App
}

    完整版的vue.js包含核心功能+模板解析器，完整版的vue在vue/dist/vue.js。

    为什么直接引入完整版的js，引入运行时的js呢？

    1. 运行时的vue文件较小，不包含模板解析器。
    2. 通过webpack打包文件的时候，webpack会帮我们解析，所以vue的模板解析器不需要打包。

    # ref属性

    • ref是用来给元素或者子组件注册引用信息(相当于HTML标签中的id，在Vue中不用直接操作id，直接操作ref)。
    • ref在html标签上获取的是真实DOM元素。
    • ref应用在组件上获取的是组件实例对象(VC)。

    来使用一个ref：

    // 在标签中使用
<h1 ref='xxx'></h1>
// 在组件中使用
<School ref='xxx'></School>

// 获取ref引用的对象
this.$refs.xxx

    # props配置项

    功能：用来让组件接收外部传过来的数据。

    把要传递的数据以组件的属性形式写出即可。

    // 父组件中使用student子组件
<Student name='李四' sex='男' :age='18' />

// 在子组件student中接收这三个参数
// 简单接收数据
props:['name','age','sex']

// 接收数据+限制类型
props：{
	name:String,
	age:Number,
	sex:String
}

// 接收数据+限制类型+默认值指定+必要性限制
props:{
	name:{
		type:String,
		required:true,
	},
	age:{
		type:Numner,
		default:99  // 默认值
	},
	sex:{
		type:String,
		required:true
	}
}

    # mixin混入

    功能：可以把多个组件公用的配置提取成一个混入对象。

    使用方法：

    // minxin.js
// 1.在minxin中定义混入。
export const mixin = {
	methods:{
		showName(){
			alert('this.name')
		}
	}
}
// School.vue
// 2.在组件中使用混入
import {mixin} from './mixin'
export default{
	name:'School',
	data(){
		return{
			name:'lynch'
		}
	},
	// 要以数组的形式进行配置
	mixins:[mixin]
}
// 全局引入
Vue.mixin(xxx)

    # 总结TodoList案例

    1. 组件化编码流程
    • 拆分静态组件：按照功能点拆分，如果命名比较困难，检查拆分是否合适，并且命名不要与html元素冲突。
    • 实现动态组件：分析数据是如何使用，当前组件使用存放在组件自身即可，一些组件共同使用，放在共同的父组件即可。
    • 绑定事件实现交互。

    1. props

      • 父组件往子组件传值。
      • 子组件往父组件传值。

      Vue对props里面的内容是浅层的监视，所以props传过来的是对象类型的值，修改对象中的属性Vue并不会报错，但是数据是单向的，所以不推荐这样做。

      所以v-mode绑定的值不能是props传过来的值，因为props里的值是不可以修改的。

    # 消息订阅与发布(pubsub)

    1. 组件间的通信方式，适用于任意组件间的通信。
    2. 使用步骤
    • 安装pubsub：npm i pubsub-js
    • 引入：import pubsub from 'pubsub-js'
    • 接收数据：A组件想要接收数据，则在A组件中订阅消息，订阅的回调函数留在A组件自身。
    methods(){
	demo(data){······}
}
// 订阅事件
mounted(){
	this.pid = pubsub.subcribe('xxx',this.demo)
}
// 解绑事件
beforeDestroy(){
	PubSub.unsubscribe(pid)
}

    # nextTick

    • 作用：在下一次DOM更新结束后执行其指定的回调。
    • 语法：this.$nextTick(回调函数)
    • 使用场景：改变数据后，要基于更新后的新DOM进行某些操作时，就可以在nextTick中指定回调函数。

    # Vue封装的过渡与动画

    1. 作用：在插入、更新或者移除DOM元素时，在合适的时候给元素添加样式类名。
    2. 写法

    元素进入的样式

    • v-enter：进入的起点
    • v-enter-active：进入过程中
    • v-enter-to：进入的终点

    元素离开的样式：

    • v-leave：离开的起点
    • v-leave-active：离开过程中
    • v-leave-to：离开的终点
    1. 用法

    使用<transiton>包裹过渡的元素，并配置name属性。

    若有多个元素需要过渡，则需要使用<transiton-group>，并且每个元素都要指定key值。

    # 使用Vue脚手架配置代理

    # 单一代理版

    devServer:{
	proxy:'http://localhost:5000'
}
    1
    2
    3
    • 优点：配置简单，请求资源直接发送即可。
    • 缺点：不能配置多个代理，不能灵活控制请求是否走代理。
    • 工作方式：当请求了前端不存在的资源时，那么该请求就会转发给服务器(优先匹配前端资源)。

    # 多个代理版

    devServer:{
	proxy:{
		'/api1':{  // 匹配所有以'/api1'开头的请求路径
				target: 'http://localhost:5000',  // 代理目标的基础路径
				changeOrigin: true,  // 设置为true，则服务器收到的请求投中的host为配置的，false为默认开启的
				pathRewrite:{'^/api1': ''}
		},
		'/api1':{
				target: 'http://localhost:5001',
				changeOrigin: true,
				pathRewrite:{'^/api2': ''}
		}
	}
}
